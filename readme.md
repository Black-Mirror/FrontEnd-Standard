## 重要
请参考[global-guide/前端项目可维护性原则.md](global-guide/前端项目可维护性原则.md)

##目录结构

 - `global/` 各种 js/html/css 全局规范
 - `apps/` 各项目及子项目的特殊规范
 - `components/` 各种组件规范

## 细节规范
  - 文件名如无特殊必要，须使用小写
  - 文件名要以命名空间为前缀，如 `ruleset-list.html`, `ruleset-update.html`
  - js, css, html 使用 2 个空格缩进
  - 注释的代码处理：要么删除，要么写上注释它的原因
